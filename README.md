# LM Studio Translate Web

一个本地翻译网页项目，直接调用你电脑上的 `LM Studio` OpenAI 兼容接口。

这个项目的目标不是“能翻译就行”，而是尽量同时做到：
- 翻译内容默认不发到外网
- 网页可在局域网内访问
- 局域网链路默认走 `HTTPS`
- 接入控制方便，但不牺牲证书信任这层安全边界

当前版本已经把旧的 `6 位 PIN` 直连方式换掉，改成：
- 远端设备先访问 `HTTPS` 页面
- 远端设备点击“发送接入申请”
- 主端网页后台批准这台设备
- 设备批准后长期访问，直到主端手动撤销

这套方式解决的是“谁能进来”，不是替代 `HTTPS` 证书信任。

## 隐私与联网边界

这个项目对 `LM Studio Base URL` 只允许这些目标：
- `127.0.0.1`
- `localhost`
- `::1`

也就是说，后端会拒绝这类 `LM Studio` 地址：
- 公网 IP
- 局域网 IP
- 任意远程域名

程序发请求时显式关闭了系统代理继承：
- `requests.Session().trust_env = False`

这样做的目的，是避免翻译请求被环境变量代理或系统代理转发到外网。

当前实现下，翻译文本流向是：
1. 浏览器把文本发给这台主机上的网页服务
2. 网页服务把文本发给本机 `LM Studio`
3. `LM Studio` 本地模型返回译文
4. 结果回到当前浏览器页面

默认不会做这些事：
- 不会把翻译文本发送到远程 API
- 不会把翻译历史写入数据库
- 不会把原文和译文持久化到项目文件里

仍然要知道的边界：
- 文本会短暂存在于当前浏览器页面内存和 Python 进程内存中
- 如果你自己把项目改成远程 `base_url`，后端会直接拒绝
- 如果 `LM Studio` 自己加载的是云端模型或云插件，那是 `LM Studio` 侧的问题，不是这个项目主动外传

## 接入安全模型

局域网接入现在分两层：

1. 证书信任
- 其他设备先信任这台主机生成的根证书
- 然后通过 `HTTPS` 访问翻译网页

2. 访问控制
- 新设备先发送接入申请
- 主端网页里出现“待审批设备”
- 主端批准一次后，这台设备长期访问
- 主端可随时撤销已批准设备

要明确一点：
- 审批后台解决的是“接入方便性”
- 根证书和 `HTTPS` 解决的是“抓包和中间人篡改”
- 这两层不能互相替代

## 功能

- 默认自动搜索本机 `LM Studio` 地址
- 默认自动生成本地根证书和 `HTTPS` 服务证书
- 主端审批后台
- 待审批设备列表
- 长期受信任设备列表
- 主端批准 / 拒绝 / 撤销设备
- 根证书下载
- 局域网 `HTTP` 快速接入页
- `Windows / macOS / Linux` 接入包下载
- `iPhone / iPad / Android` 自动识别和移动端接入入口
- 本机页面显示手机扫码接入二维码
- 本机页面动态刷新局域网地址、快速接入地址和二维码
- `HTTPS` 证书按当前局域网地址动态更新
- 读取 `LM Studio` 当前已加载模型列表
- 手动填写模型名或从列表切换模型
- 一键粘贴原文
- 一键复制原文
- 一键复制译文
- 翻译时显示进度条和状态
- 多用户按提交顺序 `FIFO` 排队翻译
- 显示排队等待信息
- 显示每次翻译本地耗时
- 动态读取显存、系统内存和 CPU 线程数，并自动调整请求侧参数
- 保存本地配置

## 默认接口

默认会自动搜索这些本机地址：
- `http://127.0.0.1:1234/v1`
- `http://localhost:1234/v1`
- `http://[::1]:1234/v1`

## 运行要求

- 已安装 `Python`
- 已安装并启动 `LM Studio`
- `LM Studio` 已开启本地 OpenAI 兼容服务
- 至少已加载一个可用于文本生成或翻译的模型

建议优先使用已经验证兼容的模型：
- `translategemma-12b-it`

某些模型即使已加载，也可能因为 `prompt template` 与当前请求格式不兼容而返回 `400` 或 `500`。这不是网页前端问题，而是模型模板本身的问题。

## 安装

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

## 启动

```powershell
.\start.ps1
```

也可以直接双击：

```text
start.bat
```

更推荐直接双击：

```text
start-hidden.vbs
```

说明：
- `start-hidden.vbs` 和 `start.bat` 都会静默启动，不保留持续显示的控制台窗口
- `start.ps1` 是底层启动脚本，供脚本调用或手动调试
- 实际服务进程通过 `pythonw.exe` 后台运行
- 启动脚本会先生成 `HTTPS` 证书，并把根证书导入当前 Windows 用户信任区，方便本机浏览器直接信任
- 启动脚本会在浏览器打开前先确认 `HTTPS` 服务已经就绪

启动后默认地址：

```text
https://127.0.0.1:7870/
```

局域网快速接入页默认地址：

```text
http://<你的局域网IP>:7871/bootstrap
```

## 局域网接入

### 主端做什么

1. 在本机打开翻译网页
2. 如有其他设备要接入，先把根证书分发给对方，或让对方打开快速接入页
3. 等远端设备打开 `HTTPS` 页面并发送接入申请
4. 在主端页面的“审批后台”里批准这台设备

### 其他电脑怎么接入

1. 先打开局域网快速接入页：

```text
http://<主机局域网IP>:7871/bootstrap
```

2. 下载适合当前系统的接入包或根证书
3. 把根证书导入系统信任区
4. 打开主端页面给出的任一 `HTTPS` 地址
5. 点击“发送接入申请”
6. 等主端网页批准

### 手机怎么接入

- 手机可直接扫描主端页面里的二维码
- 二维码会打开当前局域网快速接入页
- 页面会自动识别 `iPhone / iPad / Android`
- 导入证书后，再打开 `HTTPS` 页面发送接入申请
- 主端批准后，手机也会长期访问

## 换 Wi‑Fi 和动态地址

这个项目已经考虑了 “换 Wi‑Fi 后 IP 变化”：
- 本机页面会周期性刷新当前局域网地址
- 本机页面会同步刷新快速接入页地址和手机二维码
- `HTTPS` 服务证书会按当前局域网地址动态更新
- 如果局域网能解析主机名，也可以优先使用页面里显示的主机名地址

要注意的边界：
- 已经信任过旧证书的设备，换网络后如果地址集合变化较大，浏览器端可能需要重新打开页面
- 如果局域网里没有可用主机名解析，最终仍然要依赖当前 Wi‑Fi 下的 IP

## 多用户排队翻译

当前版本已经启用服务端 `FIFO` 队列：
- 多个用户同时提交翻译时，按提交时间顺序排队
- 同一时间只执行一个翻译任务
- 后提交的任务不会并发抢占当前模型

接口会返回这些排队信息：
- `queue_wait_ms`
- `queue_wait_seconds`
- `queue_position_at_submit`

网页消息里也会直接提示等待情况。

## 自动硬件调优

当前版本会在本地读取这些硬件信息：
- GPU 显存余量
- 系统可用内存
- CPU 线程数
- 输入文本长度

然后自动调整这些请求侧参数：
- 实际 `max_tokens`
- 分段大小
- 请求超时兜底值

页面会显示：
- 本地耗时
- 实际采用的 `Tokens`
- 当前硬件调优摘要

这里调的是“请求侧参数”，不是直接修改 `LM Studio` 的模型装载参数。

## 常见问题

### 1. 提示无法连接到 `127.0.0.1:1234`

说明 `LM Studio` 没启动，或者本地 API 服务没开。

检查：
- `LM Studio` 是否正在运行
- 本地服务端口是不是 `1234`
- `LM Studio` 里是否已启用 OpenAI 兼容接口

### 2. 提示只允许连接本机 LM Studio

这是刻意加的限制，不是 bug。

目的是确保翻译文本不会被这个项目发到外部网络。

### 3. 为什么不再用 6 位 PIN

旧版 PIN 的问题是：
- 首次接入还得人工传 PIN
- 新设备刷新或换浏览器时体验差
- 不适合长期、多设备共用

当前版本把它改成：
- 远端设备发起申请
- 主端网页审批
- 批准后长期访问

这种方式更适合多设备长期使用。

### 4. 能不能只靠审批后台，不导入证书

不能。如果你还要求：
- 局域网抓包看不到内容
- 中间人不能篡改内容

那就不能跳过证书信任这一步。

### 5. 其他设备提示证书不受信任

说明那台设备还没有信任本机生成的根证书。

处理顺序：
- 优先通过快速接入页下载当前系统对应的接入包或根证书
- 导入到系统信任区
- 再重新打开 `HTTPS` 地址

### 6. 有些模型能列出来，但翻译时报错

说明模型可能已加载，但它的 `prompt template` 和当前请求格式不兼容。

优先换成已验证模型，例如：
- `translategemma-12b-it`
