# LM Studio Translate Web

一个本地翻译网页项目，直接调用你电脑上的 `LM Studio` OpenAI 兼容接口。

这个项目的设计重点不是“能翻译就行”，而是四件事：
- 翻译内容默认不允许发到外网
- 网页可以通过局域网访问
- 局域网访问默认使用 HTTPS
- 同时尽量降低被未授权读取、抓包和中间人篡改的风险

当前局域网接入方式已经从“长访问令牌”改成更便于使用的方案：
- 首次接入输入 6 位 PIN
- 可勾选“信任这台设备 30 天”
- 后续通过受信设备会话继续使用，不需要重复输入长串令牌
- 本机页面可直接下载 `Windows 一键接入包`，其他 Windows 电脑双击导入脚本即可完成根证书安装
- macOS 和 Linux 也可以接入，但需要手动把根证书导入各自的信任存储
- 现在额外提供一个局域网 `HTTP 快速接入页`，会自动识别访问设备的系统并直接提供对应的一键接入包下载

## 隐私与联网边界

这个项目对 `LM Studio Base URL` 只允许这些目标：
- `127.0.0.1`
- `localhost`
- `::1`

也就是说，后端会拒绝这类 `LM Studio` 地址：
- 公网 IP
- 局域网 IP
- 任意远程域名

同时，程序发请求时显式关闭了系统代理继承：
- `requests.Session().trust_env = False`

这样做的目的，是避免请求被环境变量里的代理或系统代理转发出去。

当前实现下，翻译文本的流向是：
1. 浏览器把文本发给本机 Flask 服务
2. Flask 服务把文本发给本机 `LM Studio`
3. `LM Studio` 本地模型返回译文
4. 结果回到当前浏览器页面

另外，网页服务本身现在支持局域网访问，但访问端仍受这些限制：
- 只允许本机或私有网段客户端访问
- 局域网客户端首次访问必须输入 6 位 PIN
- 局域网客户端需要先导入本机根证书，再通过 HTTPS 访问
- 登录后通过 `HttpOnly + SameSite=Strict` 的受信设备会话继续使用
- 响应默认 `no-store`
- 增加 `CSP`、`X-Frame-Options`、`nosniff`、`no-referrer` 等安全头

默认不会做这些事：
- 不会把翻译文本发送到远程 API
- 不会把翻译历史写入数据库
- 不会把原文和译文持久化到项目文件里

仍然要知道的边界：
- 文本会短暂存在于当前浏览器页面内存和 Python 进程内存中
- 如果你自己把项目改成远程 `base_url`，后端会直接拒绝
- 如果 `LM Studio` 自己加载的是云端模型或云插件，那是 `LM Studio` 侧的问题，不是这个项目主动外传

最后这一点必须说清楚：
- 现在这套实现可以做到“不往外网发”“不走代理”“不接受远程 LM Studio”“局域网接入受 6 位 PIN 和受信设备会话控制”“局域网链路通过 HTTPS 加密”
- 这能显著降低同网段抓包和中间人篡改风险
- 真正前提是其他设备必须先信任本机生成的根证书；否则浏览器仍然会把证书视为不可信

## 功能

- 默认自动搜索本机 `LM Studio` 地址
- 默认自动生成本地根证书和 HTTPS 服务证书
- 6 位短 PIN 接入
- 受信设备会话
- 根证书下载
- 局域网 HTTP 快速接入页
- Windows 一键接入包下载
- macOS 一键接入包下载
- Linux 一键接入包下载
- 读取 `LM Studio` 当前已加载模型列表
- 手动填写模型名或从列表切换模型
- 一键粘贴原文
- 一键复制原文
- 一键复制译文
- 翻译时显示进度条和状态
- 保存本地配置
- 局域网接入提示
- 当前设备会话退出

## 默认接口

默认会自动搜索这些本机地址：
- `http://127.0.0.1:1234/v1`
- `http://localhost:1234/v1`
- `http://[::1]:1234/v1`

## 运行要求

- 已安装 `Python`
- 已安装并启动 `LM Studio`
- `LM Studio` 已开启本地 OpenAI 兼容服务
- 至少已加载一个可用于文本生成/翻译的模型

建议优先使用已经验证兼容的模型：
- `translategemma-12b-it`

某些模型如果在 `LM Studio` 里自带的 prompt template 不兼容，可能会返回 `400` 或 `500`。这不是网页前端问题，而是模型模板本身的问题。

## 安装

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

## 启动

```powershell
.\start.ps1
```

也可以直接双击：

```text
start.bat
```

更推荐直接双击：

```text
start-hidden.vbs
```

说明：
- `start-hidden.vbs` 和 `start.bat` 都会静默启动，不保留持续显示的控制台窗口
- `start.ps1` 是底层启动脚本，供脚本调用或手动调试
- 实际服务进程通过 `pythonw.exe` 后台运行，不再弹出 Flask 控制台
- 启动脚本会先生成 HTTPS 证书，并把根证书导入当前 Windows 用户信任区，方便本机浏览器直接信任
- 启动脚本会在浏览器打开前先确认 HTTPS 服务已经就绪，避免网页过早打开导致连接失败

启动后默认地址：

```text
https://127.0.0.1:7870/
```

局域网快速接入页默认地址：

```text
http://<你的局域网IP>:7871/bootstrap
```

如果你要让局域网内其他设备接入：
- 先在本机浏览器打开网页
- 页面顶部会显示“6 位接入 PIN”“快速接入页”“局域网接入方式”
- 其他电脑先打开局域网 `HTTP 快速接入页`
- 快速接入页会自动识别系统，并直接提供 `Windows / macOS / Linux` 对应的一键接入包下载
- 运行下载包里的导入脚本，把根证书导入本机信任区
- 其他设备打开页面顶部列出的任一 HTTPS 地址
- 首次访问时输入页面顶部显示的 6 位 PIN
- 勾选“信任这台设备 30 天”后，后续无需重复输入

## 使用说明

1. 打开 `LM Studio`
2. 加载本地模型
3. 启动本项目网页
4. 如果本机浏览器访问，直接进入翻译页
5. 如果从局域网其他设备访问，先打开 `HTTP 快速接入页`
6. 下载当前系统对应的一键接入包并导入根证书
7. 打开页面顶部显示的 HTTPS 局域网地址
8. 输入本机页面顶部显示的 6 位 PIN
9. 点击“自动搜索并刷新”
10. 选择模型或手动填写模型名
11. 粘贴原文后点击“开始翻译”

## 常见问题

### 1. 提示无法连接到 `127.0.0.1:1234`

说明 `LM Studio` 没启动，或者本地 API 服务没开。当前版本会自动搜索本机常见地址；如果仍然失败，通常就是 LM Studio 端没准备好。

检查：
- `LM Studio` 是否正在运行
- 本地服务端口是不是 `1234`
- `LM Studio` 里是否已启用 OpenAI 兼容接口

### 2. 提示只允许连接本机 LM Studio

这是刻意加的限制，不是 bug。

目的是确保翻译文本不会被这个项目发到外部网络。

### 3. 其他电脑如何直接下载适合自己系统的接入包

先打开局域网快速接入页：

```text
http://<主机局域网IP>:7871/bootstrap
```

这个页面会：
- 自动识别访问设备是 Windows、macOS 还是 Linux
- 直接给出对应的一键接入包下载按钮
- 同时保留手动切换系统包的按钮

### 4. 其他电脑提示证书不受信任

说明那台设备还没有信任本机生成的根证书。

处理顺序：
- 优先通过局域网快速接入页下载当前系统对应的一键接入包
- 或者单独下载根证书并手动导入到“受信任的根证书颁发机构”
- 重新打开 HTTPS 局域网地址

### 5. 能不能完全不用导入证书，直接更方便一点

如果你还要求：
- 局域网抓包看不到内容
- 中间人不能篡改内容

那就不能跳过“建立证书信任”这一步。

能优化的是操作路径，不是把这一步删掉。当前版本已经把它简化成：
- 其他设备先打开局域网 `HTTP 快速接入页`
- 页面自动识别系统并提供对应的一键接入包
- 运行下载包里的导入脚本
- 然后直接打开局域网 HTTPS 地址并输入 6 位 PIN

### 6. 局域网能访问，但我仍然担心内容被抓包

当前版本已经把主要风险压到这两层：
- 应用层只允许访问本机 `LM Studio`
- 传输层改为 HTTPS，并要求客户端信任本机根证书

也就是说，只要其他设备正确安装了根证书，普通局域网抓包和中间人篡改风险会显著下降。

### 7. macOS 或 Linux 能不能直接在网页里做认证，不导入证书

不能。如果你仍然要求：
- HTTPS 加密有效
- 抓包和中间人篡改风险显著下降

那浏览器在打开网页之前，就必须先信任这台主机的证书。网页里的 PIN 或登录表单只能控制“谁能访问”，不能替代“这条 HTTPS 连接是否可信”。

### 8. 有些模型能列出来，但翻译时报错

说明模型可能已加载，但它的 prompt template 和当前请求格式不兼容。

优先换成已验证模型，例如：
- `translategemma-12b-it`
