# LM Studio Translate Web

一个本地翻译网页项目，直接调用你电脑上的 `LM Studio` OpenAI 兼容接口。

这个项目的目标不是“能翻译就行”，而是尽量同时做到：
- 翻译内容默认不发到外网
- 网页可在局域网内访问
- 局域网访问默认走 HTTPS
- 尽量降低未授权读取、抓包和中间人篡改风险

## 隐私与联网边界

这个项目对 `LM Studio Base URL` 只允许这些目标：
- `127.0.0.1`
- `localhost`
- `::1`

也就是说，后端会拒绝这类 `LM Studio` 地址：
- 公网 IP
- 局域网 IP
- 任意远程域名

程序发请求时显式关闭了系统代理继承：
- `requests.Session().trust_env = False`

这样做是为了避免翻译请求被环境变量代理或系统代理转发到外网。

当前实现下，翻译文本流向是：
1. 浏览器把文本发给本机网页服务
2. 网页服务把文本发给本机 `LM Studio`
3. `LM Studio` 本地模型返回译文
4. 结果回到当前浏览器页面

默认不会做这些事：
- 不会把翻译文本发送到远程 API
- 不会把翻译历史写入数据库
- 不会把原文和译文持久化到项目文件里

仍然要知道的边界：
- 文本会短暂存在于当前浏览器页面内存和 Python 进程内存中
- 如果你自己把项目改成远程 `base_url`，后端会直接拒绝
- 如果 `LM Studio` 自己加载的是云端模型或云插件，那是 `LM Studio` 侧的问题，不是这个项目主动外传

需要明确的一点：
- 当前实现可以做到“不往外网发”“不走代理”“不接受远程 LM Studio”“局域网接入受 6 位 PIN 和受信设备会话控制”“局域网链路通过 HTTPS 加密”
- 这会显著降低同网段抓包和中间人篡改风险
- 真正前提仍然是其他设备先信任本机生成的根证书，否则浏览器依然会把证书视为不可信

## 功能

- 默认自动搜索本机 `LM Studio` 地址
- 默认自动生成本地根证书和 HTTPS 服务证书
- 6 位短 PIN 接入
- 受信设备会话
- 根证书下载
- 局域网 `HTTP` 快速接入页
- `Windows / macOS / Linux` 接入包下载
- `iPhone / iPad / Android` 自动识别和移动端接入入口
- 本机页面显示手机扫码接入二维码
- 本机页面动态刷新局域网地址、快速接入地址和二维码
- HTTPS 证书按当前局域网地址动态更新
- 读取 `LM Studio` 当前已加载模型列表
- 手动填写模型名或从列表切换模型
- 一键粘贴原文
- 一键复制原文
- 一键复制译文
- 翻译时显示进度条和状态
- 显示每次翻译的本地耗时
- 动态读取显存、系统内存、CPU 线程数，并自动下调实际请求参数
- 保存本地配置
- 当前设备会话退出

## 默认接口

默认会自动搜索这些本机地址：
- `http://127.0.0.1:1234/v1`
- `http://localhost:1234/v1`
- `http://[::1]:1234/v1`

## 运行要求

- 已安装 `Python`
- 已安装并启动 `LM Studio`
- `LM Studio` 已开启本地 OpenAI 兼容服务
- 至少已加载一个可用于文本生成或翻译的模型

建议优先使用已经验证兼容的模型：
- `translategemma-12b-it`

某些模型即使已加载，也可能因为 `prompt template` 与当前请求格式不兼容而返回 `400` 或 `500`。这不是网页前端问题，而是模型模板本身的问题。

## 安装

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

## 启动

```powershell
.\start.ps1
```

也可以直接双击：

```text
start.bat
```

更推荐直接双击：

```text
start-hidden.vbs
```

说明：
- `start-hidden.vbs` 和 `start.bat` 都会静默启动，不保留持续显示的控制台窗口
- `start.ps1` 是底层启动脚本，供脚本调用或手动调试
- 实际服务进程通过 `pythonw.exe` 后台运行，不再弹出 Flask 控制台
- 启动脚本会先生成 HTTPS 证书，并把根证书导入当前 Windows 用户信任区，方便本机浏览器直接信任
- 启动脚本会在浏览器打开前先确认 HTTPS 服务已经就绪，避免网页过早打开导致连接失败

启动后默认地址：

```text
https://127.0.0.1:7870/
```

局域网快速接入页默认地址：

```text
http://<你的局域网IP>:7871/bootstrap
```

## 局域网接入

如果要让局域网内其他设备接入：
1. 先在本机浏览器打开网页
2. 页面顶部会显示 `6 位接入 PIN`、局域网 HTTPS 地址、快速接入页地址和手机二维码
3. 其他设备先打开局域网 `HTTP` 快速接入页
4. 快速接入页会自动识别设备类型，并给出对应下载入口
5. 先导入根证书，再打开 HTTPS 局域网地址
6. 首次访问输入本机页面显示的 `6 位 PIN`
7. 勾选“信任这台设备 30 天”后，后续无需重复输入

### 桌面设备

- `Windows`：优先使用一键接入包
- `macOS`：下载接入包或根证书后，导入系统信任区
- `Linux`：下载接入包或根证书后，导入系统信任区

### 手机设备

- 手机可直接扫描本机页面上的“手机接入”二维码
- 二维码会打开当前局域网快速接入页
- 快速接入页会自动识别 `iPhone / iPad / Android`
- `iPhone / iPad`：优先提供 `.mobileconfig`
- `Android`：优先提供根证书 `.crt`
- 导入完成后，再打开页面里给出的 HTTPS 地址并输入 `6 位 PIN`

## 动态地址与换 Wi‑Fi

这个项目已经考虑了“换 Wi‑Fi 后 IP 变化”的情况：
- 本机页面会周期性刷新当前局域网地址
- 本机页面会同步刷新快速接入页地址和手机二维码
- HTTPS 服务证书会按当前局域网地址动态更新
- 如果你的手机或其他设备能解析主机名，也可以优先使用页面中显示的主机名地址，减少对 IP 变化的依赖

要注意的边界：
- 已经信任过旧证书的设备，切换网络后如果地址集合变化较大，浏览器端可能需要重新打开页面
- 如果局域网里没有可用的主机名解析，最终仍然要依赖当前 Wi‑Fi 下的 IP

## 使用说明

1. 打开 `LM Studio`
2. 加载本地模型
3. 启动本项目网页
4. 如果本机浏览器访问，直接进入翻译页
5. 如果从局域网其他设备访问，先打开快速接入页或扫码
6. 导入根证书
7. 打开页面顶部显示的 HTTPS 地址
8. 输入本机页面顶部显示的 `6 位 PIN`
9. 点击“自动搜索并刷新”
10. 选择模型或手动填写模型名
11. 粘贴原文后点击“开始翻译”

## 自动硬件调优

当前版本会在本地读取这些硬件信息：
- GPU 显存余量
- 系统可用内存
- CPU 线程数
- 输入文本长度

然后自动调整这些请求侧参数：
- 实际 `max_tokens`
- 分段大小
- 请求超时兜底值

页面会显示：
- 本地耗时
- 实际采用的 `Tokens`
- 当前硬件调优摘要

这部分调的是“请求侧参数”，不是直接修改 `LM Studio` 的模型装载参数。也就是说，它能让当前已加载模型跑得更稳，但不能替你切换量化版本或改 `LM Studio` 内部的 GPU offload 设置。

## 常见问题

### 1. 提示无法连接到 `127.0.0.1:1234`

说明 `LM Studio` 没启动，或者本地 API 服务没开。

检查：
- `LM Studio` 是否正在运行
- 本地服务端口是不是 `1234`
- `LM Studio` 里是否已启用 OpenAI 兼容接口

### 2. 提示只允许连接本机 LM Studio

这是刻意加的限制，不是 bug。

目的是确保翻译文本不会被这个项目发到外部网络。

### 3. 其他设备如何下载适合自己的接入包

先打开快速接入页：

```text
http://<主机局域网IP>:7871/bootstrap
```

这个页面会自动识别：
- `Windows`
- `macOS`
- `Linux`
- `iPhone / iPad`
- `Android`

并直接给出对应下载入口。

### 4. 其他设备提示证书不受信任

说明那台设备还没有信任本机生成的根证书。

处理顺序：
- 优先通过快速接入页下载对应设备的接入包或证书
- 导入到系统信任区
- 重新打开 HTTPS 局域网地址

### 5. 能不能完全不用导入证书，直接更方便一点

如果你还要求：
- 局域网抓包看不到内容
- 中间人不能篡改内容

那就不能跳过“建立证书信任”这一步。

能优化的是操作路径，不是把这一步删掉。当前版本已经把它压缩成：
- 打开快速接入页
- 自动识别设备
- 下载对应安装包或证书
- 导入后直接打开 HTTPS 页面

### 6. 局域网能访问，但我仍然担心内容被抓包

当前版本已经把主要风险压到这两层：
- 应用层只允许访问本机 `LM Studio`
- 传输层改为 HTTPS，并要求客户端信任本机根证书

也就是说，只要其他设备正确安装了根证书，普通局域网抓包和中间人篡改风险会显著下降。

### 7. 手机能不能只靠网页认证，不导入证书

不能。

如果仍然要求：
- HTTPS 加密有效
- 抓包和中间人篡改风险显著下降

那浏览器在打开网页之前，就必须先信任这台主机的证书。网页里的 PIN 或登录表单只能控制“谁能访问”，不能替代“这条 HTTPS 连接是否可信”。

### 8. 有些模型能列出来，但翻译时报错

说明模型可能已加载，但它的 `prompt template` 和当前请求格式不兼容。

优先换成已验证模型，例如：
- `translategemma-12b-it`
