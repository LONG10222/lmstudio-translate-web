# LM Studio Translate Web

一个本地翻译网页项目，直接调用你电脑上的 `LM Studio` OpenAI 兼容接口。

这个项目的设计重点不是“能翻译就行”，而是三件事：
- 翻译内容默认不允许发到外网
- 网页可以通过局域网访问
- 局域网访问尽量降低被未授权读取或劫持的风险
- 操作尽量短路径，适合频繁复制、粘贴、翻译

## 隐私与联网边界

这个项目对 `LM Studio Base URL` 只允许这些目标：
- `127.0.0.1`
- `localhost`
- `::1`

也就是说，后端会拒绝这类 `LM Studio` 地址：
- 公网 IP
- 局域网 IP
- 任意远程域名

同时，程序发请求时显式关闭了系统代理继承：
- `requests.Session().trust_env = False`

这样做的目的，是避免请求被环境变量里的代理或系统代理转发出去。

当前实现下，翻译文本的流向是：
1. 浏览器把文本发给本机 Flask 服务
2. Flask 服务把文本发给本机 `LM Studio`
3. `LM Studio` 本地模型返回译文
4. 结果回到当前浏览器页面

另外，网页服务本身现在支持局域网访问，但访问端仍受这些限制：
- 只允许本机或私有网段客户端访问
- 局域网客户端首次访问必须输入访问令牌
- 局域网会话使用 `SameSite=Strict`、`HttpOnly` Cookie
- 响应默认 `no-store`
- 增加 `CSP`、`X-Frame-Options`、`nosniff`、`no-referrer` 等安全头

默认不会做这些事：
- 不会把翻译文本发送到远程 API
- 不会把翻译历史写入数据库
- 不会把原文和译文持久化到项目文件里

仍然要知道的边界：
- 文本会短暂存在于当前浏览器页面内存和 Python 进程内存中
- 如果你自己把项目改成远程 `base_url`，后端会直接拒绝
- 如果 `LM Studio` 自己加载的是云端模型或云插件，那是 `LM Studio` 侧的问题，不是这个项目主动外传
- 如果你的局域网本身不可信，单纯 HTTP 仍然不能提供密码学级别的防窃听和防篡改

最后这一点必须说清楚：
- 现在这套实现可以做到“不往外网发”“不走代理”“不接受远程 LM Studio”“局域网访问受令牌保护”
- 但如果你要求“同网段有人抓包也完全看不到、也完全改不了”，那就必须再加 HTTPS 或 VPN
- 也就是说，当前版本已经把应用层外传风险压住了，但不能虚构成等同于加密隧道

## 功能

- 默认自动搜索本机 `LM Studio` 地址
- 读取 `LM Studio` 当前已加载模型列表
- 手动填写模型名或从列表切换模型
- 一键粘贴原文
- 一键复制原文
- 一键复制译文
- 翻译时显示进度条和状态
- 保存本地配置
- 局域网访问令牌
- 局域网会话退出

## 默认接口

默认会自动搜索这些本机地址：
- `http://127.0.0.1:1234/v1`
- `http://localhost:1234/v1`
- `http://[::1]:1234/v1`

## 运行要求

- 已安装 `Python`
- 已安装并启动 `LM Studio`
- `LM Studio` 已开启本地 OpenAI 兼容服务
- 至少已加载一个可用于文本生成/翻译的模型

建议优先使用已经验证兼容的模型：
- `translategemma-12b-it`

某些模型如果在 `LM Studio` 里自带的 prompt template 不兼容，可能会返回 `400` 或 `500`。这不是网页前端问题，而是模型模板本身的问题。

## 安装

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

## 启动

```powershell
.\start.ps1
```

也可以直接双击：

```text
start.bat
```

更推荐直接双击：

```text
start-hidden.vbs
```

说明：
- `start-hidden.vbs` 和 `start.bat` 都会静默启动，不保留持续显示的控制台窗口
- `start.ps1` 是底层启动脚本，供脚本调用或手动调试
- 实际服务进程现在通过 `pythonw.exe` 后台运行，不再弹出 Flask 控制台

启动后默认地址：

```text
http://127.0.0.1:7870/
```

如果你要让局域网内其他设备接入：
- 先在本机浏览器打开网页
- 页面顶部会显示“局域网接入方式”
- 其他设备打开其中任一地址
- 首次访问时输入页面顶部显示的“局域网访问令牌”

## 使用说明

1. 打开 `LM Studio`
2. 加载本地模型
3. 启动本项目网页
4. 如果本机浏览器访问，直接进入翻译页
5. 如果从局域网其他设备访问，先输入本机页面顶部显示的访问令牌
6. 点击“自动搜索并刷新”
7. 选择模型或手动填写模型名
8. 粘贴原文后点击“开始翻译”

## 常见问题

### 1. 提示无法连接到 `127.0.0.1:1234`

说明 `LM Studio` 没启动，或者本地 API 服务没开。当前版本会自动搜索本机常见地址；如果仍然失败，通常就是 LM Studio 端没准备好。

检查：
- `LM Studio` 是否正在运行
- 本地服务端口是不是 `1234`
- `LM Studio` 里是否已启用 OpenAI 兼容接口

### 2. 提示只允许连接本机 LM Studio

这是刻意加的限制，不是 bug。

目的是确保翻译文本不会被这个项目发到外部网络。

### 3. 局域网能访问，但我仍然担心内容被抓包

这个担心是合理的。

当前版本防的是：
- 外网转发
- 代理转发
- 未授权局域网访问
- 常见浏览器级劫持

当前版本不能单独解决的是：
- 不可信局域网里的被动抓包
- 不可信局域网里的中间人篡改

这两类风险如果你也要一起压住，需要继续加：
- HTTPS
- 或 VPN / 零信任内网通道

### 4. 有些模型能列出来，但翻译时报错

说明模型可能已加载，但它的 prompt template 和当前请求格式不兼容。

优先换成已验证模型，例如：
- `translategemma-12b-it`
