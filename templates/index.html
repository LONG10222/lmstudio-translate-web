Windows

优先使用一键接入包。导入根证书后，直接打开 HTTPS 地址并点击“发送接入申请”。

macOS

导入根证书到钥匙串并设为始终信任，然后打开 HTTPS 地址提交接入申请。

sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain lmstudio-translate-root-ca.crt
Linux

先把根证书导入系统证书存储，再访问 HTTPS 地址提交接入申请。Firefox 可能还需要单独导入。

sudo cp lmstudio-translate-root-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates
显示隐私约束

翻译请求只会从当前网页发到这台主机，再由这台主机发给本机 `LM Studio`。后端会拒绝公网、域名和局域网形式的远程 `LM Studio` 地址。

接入审批只是访问控制。真正防抓包和中间人篡改的前提，仍然是其他设备先信任这台主机生成的根证书。
