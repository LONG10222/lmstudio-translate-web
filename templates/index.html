Windows

优先使用一键接入包。解压后运行 Install-RootCA.ps1，再打开上面的 HTTPS 地址。

macOS

下载根证书后导入钥匙串，并把它设为“始终信任”。也可以用命令行导入系统信任区。

sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain lmstudio-translate-root-ca.crt
Linux

先导入系统证书存储，再访问 HTTPS 地址。Firefox 在部分发行版上可能还要单独导入证书。

sudo cp lmstudio-translate-root-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates
显示隐私约束

翻译请求只会从网页发到这台电脑，再由这台电脑发给本机 LM Studio。后端会拒绝公网、域名和局域网形式的 LM Studio 地址。

如果其他电脑要通过局域网访问，为了防抓包和中间人篡改，仍然需要先信任这台主机生成的根证书。网页 PIN 只能控制访问权限，不能替代 TLS 证书信任。

为什么不能只靠网页认证

因为浏览器在建立安全连接之前，就必须先判断这台主机的证书是否可信。只靠网页里的 PIN 或登录表单，不能解决抓包和中间人篡改问题。
